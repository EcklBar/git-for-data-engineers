Introduction to Git

What is Version Control?

  • Stores and manages every revision of files and code
  • Collaborate on source code & manage releases
  • Roll back to previous versions when bugs are discovered
  • VSC = Version Control System

About Git

  • Created by Linus Torvalds in 2005 (for the Linux Kernel)
  • Replaces older VCS tooling (CVS, Subversion, BitKeeper, Mercurial)
  • Focuses on source code distribution (everyone has a copy)
  • Implements efficient file storage
  • Ensures files integrity

Git Advantages

  • Distributed development (work on your local copy)
  • High performance
  • Ease of use
  • Strong support for branching and history management
  • Extensive tooling for advanced operations

Git Challenges

  • A new code management model
  • Learning curve (mind shift & commands)
  • Destructive operations when used incorrectly
  • Limited support for binary files

Commit & Push
  • Get a copy of the entire repository from the server ("clone")
  • All the content and all the history
  • Make changes locally (commits and branches)
  • Push changes back to the server, a.k.a. remote repository

Git Object Types

  • Blob: The raw content of a file ‚Äì without file metadata
  • Tree: One level of directory information (file names, blobs, subdirectories)
  • Commit: The version of the tree + metadata (author, dates, log messages, etc.)
  • Tag: A label associated with a commit, e.g., a version number

Delta Storage

  • Each file is stored as a complete, separate object when introduced
  • Changes to an existing file are stored as deltas
  • Git uses a smart and efficient mechanism for storing changes
  • Results in significant space savings and efficient history management

Git Workflow and Three Stages

üìÅ The Three Stages
Working Directory

Working Directory

  • Create, edit, and delete files in your project
  • Think of it as your "sandbox" where you work on your code and data

Staging Area (Index)

  • A checklist of changes you want to commit
  • Prepare and select specific changes to be included in the next commit
  • Using git add, you move changes from working directory to staging area

Repository

  • The repository stores your project's complete history
  • When you git commit, the changes in the staging area are recorded as a new snapshot in the repository
  • Your project's complete history is stored here

File States

  • Untracked: A new file that is first introduced
  • Unmodified: An existing file with no local changes
  • Modified: An existing file with local changes
  • Staged: Files with local changes marked for the next commit
  • Committed: Changes safely stored in the local database

Git Branches

üå≥ Branching Strategy
  • Treat the source code as a tree of branches
  • Each branch is a linear set of commits
  • Use a side branch to work on a private version of the code
  • Merge the side branch back into the main branch
  • Collaborate with other developers
git checkout branch_name
- Switch to branch
git checkout -B branch_name
- Create and switch to new branch
git branch -D testing
- Delete branch
git log --oneline --decorate --graph --all
- View all branches
git show branch_name
- Show branch head

Merge Branches

  • Once ready, a side branch is merged into the main branch
  • Switch to the main branch: git checkout main
  • Merge the side branch: git merge side_branch

Divergent History

  • Collaborating using multiple branches may lead to divergent history
  • Developer #1 starts a new branch and changes a file
  • Developer #2 adds a new commit to the main branch and also changes the same file
  • Developer #1 cannot merge because the changes conflict

Merge Conflicts

‚ö° Facing Conflicts
  • When a merge conflict emerges, the merge is paused to let you resolve it
  • Use git status to see which files have conflicts
  • Files without conflicts are staged
  • Files with conflicts appear as "Both modified"
Conflict Example:
<<<<<<< HEAD
new text on main
=======
text from side branch
>>>>>>> my_side_branch

Resolving Conflicts Manually

  • Open the file that has conflicts
  • Delete/Change irrelevant lines
  • Stage the file (git add)
  • To finish the merge when all conflicts are resolved: git merge --continue

Resolving With Style

  • Modern IDE viewing and resolving merge conflicts (e.g., VSCode)
  • Using a merge editor is the better practice
  • See the changes more clearly
  • Easily choose the preferred version
  • Combine code from both branches
  • More intuitive and human-friendly interface
üîß VSCode Merge Example
  1. Open the file in conflict
  2. Press "Resolve in Merge Editor"
  3. Compare the changes and resolve the conflict
  4. Choose "Accept Incoming" to select changes from the side branch
  5. Choose "Accept Current" to select changes from the main branch
  6. Manually update the result if needed
  7. Once ready, press "Complete Merge" to stage the file
  8. Run git merge --continue or use VSCode to finish

Essential Git Commands

Repository Management

git init
- Create a new local repository
git clone
- Download a copy of a remote repository
git status
- Show the status of the working directory
git remote -v
- List all remote repositories including their URL(s).
git remote
- List all remote repositories
git remote add origin <url>
- Add a new remote repository
git remote add new_name <path>
- Add a new name for the remote repository

Staging & Committing

git add <file>
- Stage a specific file
git add .
- Stage all changes in the current directory
git restore --staged <file>
- Remove a file from staging (same as reset for that file)
git restore --staged
- Remove all files from staging
git reset
- Unstage files
git reset --hard
- Reset working directory to the last commit
git commit
- Create a new commit (changeset) in the local repository
git commit -m "message"
- Commit with an inline message

Branching & Merging

git branch
- List all branches
git branch <name>
- Create a new branch
git branch -d <name>
- Delete a local branch (prevents deletion if there are unmerged changes)
git branch -D <name>
- Force delete a local branch (even if there are unmerged changes)
git branch -m old_name new_name
- Renameing a branch
git checkout <branch>
- Switch to a different branch
git checkout -b <name>
- Create and switch to a new branch (fails if branch exists)
git checkout -B <name>
- Create or reset an existing branch to the current HEAD
git merge <branch>
- Merge a branch into the current branch
git merge <branch> --ff-only
- Merge only if fast-forward is possible
git merge --no-ff <branch>
- Always create a merge commit, even if fast-forward is possible
git merge --squash <branch>
- Combine all commits from the branch into a single commit without creating a merge commit
git merge -s octopus <branch1> <branch2> ...
- Merge multiple branches into the current branch with a single merge commit

Cherry-pick

git cherry-pick <commit-hash>
- Apply a commit from another branch
git cherry-pick <hash1> <hash2>
- Apply multiple commits
git cherry-pick --continue
- Continue after resolving conflicts
git cherry-pick --abort
- Cancel cherry-pick process

History & Information

git log
- View the history of commits in the local repository
git log --oneline
- Compact commit history
git log --oneline --graph --all
- Compact commit history with a visual graph of all branches
git log --since="YYYY-MM-DD" --until="YYYY-MM-DD"
- View commit history within a date range
git show <commit>
- Show details of a specific commit
git diff
- Compare changes (local, staged, or between commits)
git diff --staged
- Compare staged files to the last commit
git blame <file>
- Show who modified each line in a file
git tag <tag-name>
- Create a local tag for the current commit to mark an important version

Undoing Changes

git checkout <file>
- Restore a file to its last committed state
git revert <commit>
- Create a new commit that reverts changes from a specific commit
git revert <commit> --no-edit
- Revert without opening a text editor
git revert <commit> -n
- Revert without creating a new commit
git reset HEAD <file>
- Unstage a specific file
git reset --soft HEAD~1
- Undo last commit but keep changes staged
git reset --mixed HEAD~1
- Undo last commit, keep changes in working directory but unstaged
git reset --hard HEAD~1
- Undo last commit and discard changes

Rebase

git rebase -i master
- Interactively rebase the current branch onto master, allowing you to edit, combine, or remove commits.
  1. p, pick = use commit as is
  2. r, reword = use commit, but edit the commit message
  3. e, edit = use commit, but pause for amending
  4. s, squash = combine commit into the previous one (keep both messages)
  5. f, fixup = combine commit into the previous one (discard this commit's message)
  6. x, exec = run a shell command during rebase
  7. d, drop = remove commit entirely

Bisect

git bisect
- Start binary search to find a bug
git bisect start
- Begin the bisect session
git bisect bad
- Mark current commit as bad
git bisect good <commit-hash>
- Mark a known good commit
git bisect run <script_name>
- Automate testing with a script
git bisect reset
- End bisect and return to original state

Filter Repo

git install git-filter-repo
- Install filter-repo tool
git filter-repo --path secret.txt --invert-paths
- Remove specific file from history
git filter-repo --path config.ini --invert-paths --force
- Force remove file from history

Reflog

git reflog
- Show reference log
git reflog show
- Show reflog details
git reflog expire
- Clean up old reflog entries
git reflog --since
- Show reflog entries since a given time
git reflog --until
- Show reflog entries until a given time

Worktree

git worktree
- Manage multiple working directories
git worktree add <path> <branch>
- Create a new working tree at a path for a branch
git worktree list
- Show all linked working trees
git worktree remove <path>
- Remove a working tree

Git Submodule

git submodule
- Manage submodules
git submodule add <repository link | dir> <path>
- Add a new submodule
git submodule list
- Show submodules
git submodule status
- Show the status (commit + sync state) of submodules
git submodule update --init
- Initialize and fetch submodules
git submodule update --init --recursive
- Initialize and update all nested submodules
git submodule update --init <path_to_submodule>
- Init & update specific submodule
git submodule deinit <submodule_name>
- Unregister a submodule
git rm <path>
- Remove a submodule from repo
